# Conformance report JSON

Run the suite against a server with:

```sh
cd conformance
npx conformance run --against http://localhost:9657 --report json
```

The command writes one JSON document to stdout. A failed conformance run still
emits its report and exits non-zero, which lets CI retain the measurement while
using the exit status as its gate.

## Format

```json
{
  "schemaVersion": 1,
  "against": "http://localhost:9657",
  "results": [
    {
      "id": "SEMANTICS-BOUNDED-INTEGER-001",
      "name": "...",
      "suite": "suites/semantics/bounded-integer.test.ts",
      "status": "passed",
      "durationMs": 4.2
    }
  ],
  "summary": {
    "total": 1,
    "passed": 1,
    "failed": 0,
    "skipped": 0,
    "todo": 0
  }
}
```

`id` is the stable identifier used by CI and experiment scoring. `status` is
one of `passed`, `failed`, `skipped`, or `todo`. Failed results include an
`error` string; successful results omit it. Results are sorted by `id` so
reports are stable regardless of Vitest file execution order.
